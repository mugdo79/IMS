# 핵심 원칙 (Core Beliefs)

이 문서는 디지털 병리 IMS 프로젝트에서 기술적 판단을 내릴 때 기준이 되는 핵심 원칙을 정리합니다.
구체적인 아키텍처 결정은 [`ARCHITECTURE.md`](../../ARCHITECTURE.md)를 참고하고, 이 문서는 "왜 그렇게 결정했는가"의 배경이 되는 더 상위의 가치 판단을 다룹니다.

## 1. 진단 정확성 > 다른 모든 것

이 시스템의 존재 이유는 판독의가 정확하게 진단하도록 돕는 것입니다. 성능, 비용, 개발 속도는 진단 정확성을 해치지 않는 선에서만 의미가 있습니다.

- "기술적으로 동작한다"는 합격 기준이 아닙니다. "판독의가 실제로 이걸로 판독할 수 있는가"가 기준입니다.
- 예: 타일 서버를 OpenSlide 단독 구성에서 OpenSlide+pyvips로 전환한 결정 — 속도/리소스가 아니라 판독의 검증 결과(세포 구분 시 타일 비선명)가 트리거였음 ([`ARCHITECTURE.md`](../../ARCHITECTURE.md) 결정 #4).
- 이미지 품질, 좌표/스케일(MPP) 정확성, 어노테이션 데이터 무결성과 관련된 변경은 "동작 확인"만으로 끝내지 않고 가능하면 임상적 관점에서 재확인을 거칩니다.

## 2. 환자 데이터 보호와 감사 가능성은 기본값이다

PHI/PII 처리와 접근 기록은 나중에 추가하는 보안 기능이 아니라, 처음부터 아키텍처에 내장되어 있어야 합니다.

- 모든 환자 데이터 접근은 `AuditLog`로 남습니다 (세션 단위 — [`ARCHITECTURE.md`](../../ARCHITECTURE.md) 결정 #2).
- 새 기능을 추가할 때 "이 기능이 환자 데이터에 접근하는가? 그렇다면 권한 검증과 감사로그가 빠짐없이 들어가는가?"를 기본 체크리스트로 삼습니다.
- 편의를 위해 인증/감사를 우회하는 임시 코드(디버그 백도어, 테스트용 전체 권한 등)는 만들지 않습니다.

## 3. 비용이 큰 결정은 일찍, 나머지는 점진적으로

스키마/데이터 모델처럼 나중에 바꾸기 어려운 결정은 처음부터 전체 도메인 깊이를 반영해서 설계하되, UI나 기능 노출은 점진적으로 진행합니다.

- 예: 데이터 모델은 `Case → Specimen → Block → Slide` 4단계 전체를 처음부터 반영하지만, 초기 UI에서는 단순화해서 보여줄 수 있음.
- 반대로 당장 필요 없는 유연성을 위한 추상화(premature abstraction)는 만들지 않습니다 — 단, "온프레미스 전환 가능성"처럼 이 프로젝트의 맥락에서 현실적인 시나리오는 예외입니다 (스토리지 추상화 레이어).

## 4. 솔로 개발 환경의 운영 부담을 최소화한다

인프라/운영 복잡도는 그 자체로 리스크입니다. 솔로 개발자가 감당할 수 있는 수준으로 구성요소 수를 최소화합니다.

- 예: NoSQL 도입 없이 PostgreSQL+PostGIS 단일 RDB로 운영 ([`ARCHITECTURE.md`](../../ARCHITECTURE.md) 결정 #3).
- 새로운 인프라 컴포넌트(메시지 큐, 캐시 서버, 별도 DB 등)를 추가하기 전에 "기존 구성요소로 충분한가"를 먼저 검토합니다.

## 5. AI는 보조 도구이며, 최종 판단은 판독의의 몫이다

AI가 생성한 데이터(`CellAnnotation`)와 판독의가 직접 작성한 데이터(`Annotation`)는 명확히 구분되어 저장/표시됩니다.

- AI 결과는 참고 정보로 제공되며, 판독의의 판독을 대체하거나 자동으로 진단 결과에 반영되지 않습니다.
- UI에서도 두 종류의 어노테이션은 출처가 명확히 구분되어야 합니다 (예: 다른 스타일/레이어).

## 6. 서비스 경계를 가로지르는 변경은 영향 범위를 먼저 확인한다

이 시스템은 여러 독립 서비스(frontend/backend/tile-server)로 구성되어 있고, 일부 결정(토큰 기반 통신, 감사 단위 등)은 서비스 경계를 가로지릅니다.

- 한 서비스의 변경이 다른 서비스와의 계약(토큰 포맷, API, 감사 이벤트 등)에 영향을 주는지 항상 확인합니다.
- 영향 범위가 불명확하거나 여러 서비스에 걸친 변경은 진행 전에 [`ARCHITECTURE.md`](../../ARCHITECTURE.md)에 먼저 기록하고 합의합니다.
