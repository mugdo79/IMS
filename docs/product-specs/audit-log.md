# 감사로그 (Audit Log)

`AuditLog`에 기록되는 이벤트의 종류, 데이터 구조, 조회 권한, 보존 정책을 정의합니다. 감사가능성은 [`core-beliefs.md`](../design-docs/core-beliefs.md) 원칙 #2의 핵심 요구사항입니다.
다른 스펙에서 이미 정의한 감사 이벤트(상태 전이, 어노테이션 변경, 열람 세션)를 이 문서에서 통합 정리합니다.

## 데이터 구조

| 필드 | 설명 |
|---|---|
| 사용자 | 행위자 (`User` 참조) |
| 액션 | 이벤트 타입 (아래 표 참고) |
| 대상 리소스 | 관련 엔티티 타입 + ID (예: `Case#123`, `Annotation#456`) |
| 타임스탬프 | 발생 시각 |
| IP | 요청 IP |
| 상세 (JSONB) | 이벤트별 부가 정보 (이전/이후 값, 사유 등) |

`AuditLog`는 추가만 가능(append-only)하며 수정·삭제되지 않습니다.

## 기록되는 이벤트

| 이벤트 | 발생 시점 | 상세 내용 |
|---|---|---|
| 슬라이드 열람 (세션 시작) | 뷰어 토큰 발급 시 | Slide ID, Case ID ([`ARCHITECTURE.md`](../../ARCHITECTURE.md) 결정 #2) |
| 슬라이드 열람 (세션 종료) | [세션 종료 기록](#세션-종료-기록) 참고 | 세션 길이 |
| Case 상태 전이 | 상태 변경 시 | 이전 상태 → 이후 상태 ([`case-status-workflow.md`](case-status-workflow.md)) |
| 재오픈 | 서명완료 → 검토 | 재오픈 사유 |
| 담당 판독의 배정 / 재배정 | 배정 변경 시 | 이전 담당자 → 신규 담당자 |
| Annotation 생성 / 수정 / 삭제 | 어노테이션 변경 시 | 변경 전/후 지오메트리·타입·측정값 스냅샷 ([`annotation.md`](annotation.md)) |
| 환자 / 케이스 / 검체 / 블록 / 슬라이드 정보 등록·수정 | 등록/수정 시 | 변경 필드별 이전/이후 값 |

"검토 → 서명완료" 전이 시에는 `Case.진단 소견`의 내용도 상세(JSONB)에 함께 기록됩니다.

## 세션 종료 기록

- 세션 시작은 토큰 발급 시점으로 명확하지만, 종료는 frontend의 명시적 "닫기" 이벤트에 의존하지 않습니다 (탭을 그냥 닫는 경우가 흔함).
- frontend는 뷰어가 열려있는 동안 주기적으로(예: 1분 간격) backend에 heartbeat를 보내고, backend는 해당 세션의 `AuditLog` 레코드에 "마지막 활동 시각"을 갱신합니다.
- 마지막 활동 시각 이후 일정 시간(heartbeat 간격의 2~3배) 갱신이 없으면 세션이 종료된 것으로 간주합니다.
- 이 heartbeat는 [`wsi-viewer.md`](wsi-viewer.md)의 토큰 갱신 요청과 결합될 수 있습니다 — 구체적 구현은 [`ARCHITECTURE.md`](../../ARCHITECTURE.md) 토큰 갱신 흐름 설계에서 다룹니다.

## 조회 / 열람 권한

| 조회 범위 | 가능한 역할 | 위치 |
|---|---|---|
| 특정 Case의 이력 (상태전이/배정/어노테이션/열람) | 해당 Case에 접근 가능한 모든 역할 | Case 상세 화면의 "이력" 탭 |
| 시스템 전체 AuditLog (사용자/환자 기준 검색 등) | 관리자 | 별도 감사 화면 |

## 보존 정책

- 스토리지/비용 추정을 위해 최소 5년 이상 보존을 기본 가정으로 합니다 (의료 기록 보존 요구사항에 준함). 정확한 기간은 법률 검토 후 확정합니다.
- 보존 기간 내에는 수정/삭제 불가 (append-only).

## 미해결 / 추후 논의

- `AuditLog` 보존 기간 — 의료법상 진료기록 보존 요구사항 확인 필요
- 로그인/로그아웃 등 인증 이벤트 기록 여부 — 인증 방식 자체가 미결정 ([`ARCHITECTURE.md`](../../ARCHITECTURE.md))
- 세션 종료 heartbeat의 구체적 구현 (간격, 토큰 갱신과의 결합 방식)
- Case/환자 정보를 뷰어 없이 목록·상세에서만 조회한 경우도 감사 대상에 포함할지
- 관리자용 "시스템 전체 AuditLog" 화면의 검색/필터 UX
