# WSI 뷰어

WSI 뷰어 화면의 구성, 타일 로딩 방식, 레이어 구성, 어노테이션 도구, 권한별 동작을 정의합니다.
타일 서빙/토큰 흐름은 [`ARCHITECTURE.md`](../../ARCHITECTURE.md) 결정 #2, 어노테이션 데이터 구조는 [`annotation.md`](annotation.md), Case 상태별 권한은 [`case-status-workflow.md`](case-status-workflow.md) 참고.

## 기본 동작

- OpenSeadragon 기반으로 DZI 타일을 tile-server에서 토큰으로 직접 요청합니다.
- 초기 뷰는 슬라이드 전체(라벨/스캔 영역 포함)가 보이는 최저 배율로 시작합니다.
- 줌/팬은 표준 OpenSeadragon 인터랙션을 사용합니다.
- 현재 줌 레벨을 `Slide.MPP`로 환산한 배율(예: "20.0x")로 화면에 표시합니다.

## 슬라이드 간 이동

- 한 `Case`는 여러 `Slide`를 포함하므로, 뷰어 내 슬라이드 목록(썸네일 패널)에서 전환할 수 있습니다.
- 목록에는 각 슬라이드의 썸네일/라벨 이미지를 표시합니다 — 슬라이드 오인 방지 ([`core-beliefs.md`](../design-docs/core-beliefs.md) 원칙 #1).
- 슬라이드를 전환하면 뷰포트(줌/위치)는 해당 슬라이드의 기본값(전체 보기)으로 초기화됩니다.
- MVP는 한 번에 하나의 슬라이드만 표시합니다 — 다중 슬라이드 동시 비교는 향후 범위입니다 ([`PRODUCT_SENSE.md`](../../PRODUCT_SENSE.md)).

## 레이어 구성

| 레이어 | 내용 | 기본 표시 | 토글 | 편집 |
|---|---|---|---|---|
| 베이스 (WSI 타일) | tile-server가 서빙하는 DZI 타일 | 항상 | - | - |
| Annotation | 판독의가 작성한 영역/측정/메모 | ON | 가능 | "검토" 상태에서만 ([`case-status-workflow.md`](case-status-workflow.md)) |
| CellAnnotation (AI) | AI 형태계측 결과 | **OFF** | 가능 | 읽기 전용 |

- `CellAnnotation` 레이어를 기본 OFF로 두는 것은, 판독의가 먼저 직접 판독한 뒤 필요 시 AI 결과를 참고하도록 하기 위함입니다 — "AI 보조와 판독의 판단의 분리" ([`PRODUCT_SENSE.md`](../../PRODUCT_SENSE.md) Product Sense 기준, [`core-beliefs.md`](../design-docs/core-beliefs.md) 원칙 #5).
- 레이어 간 시각적 구분 스타일은 [`annotation.md`](annotation.md) 참고.

## 어노테이션 도구

- 그리기 도구: 영역(폴리곤), 측정(선/폐곡선), 메모(포인트+텍스트) — 타입 정의는 [`annotation.md`](annotation.md) 참고.
- 측정 도구는 드래그 중 실시간으로 실제 단위 값을 표시합니다.
  - 길이: μm 단위로 표시
  - 면적: μm² 단위로 표시하되, 1,000,000 μm²(=1mm²) 이상이면 mm²로 자동 전환
- 도구 활성화 여부는 `Case.상태`에 따릅니다 — "검토" 상태에서만 작성/수정/삭제 가능하며, 그 외 상태에서는 도구가 비활성화되고 기존 어노테이션은 읽기 전용으로 표시됩니다.

## 어노테이션 목록

- 현재 슬라이드에 작성된 `Annotation` 목록을 패널로 표시합니다 (타입, 작성자, 측정값 요약).
- 목록 항목을 클릭하면 뷰어가 해당 어노테이션 위치로 이동(팬+줌)합니다.

## 이미지 보정 도구

- **감마 / 밝기 / 명암**: 클라이언트(브라우저) 측 Canvas/WebGL 필터로 적용됩니다. 슬라이더 조작 시 재요청 없이 즉시 반영됩니다.
- **ICC 프로필 적용/미적용**: 슬라이드의 ICC 프로필로부터 추출한 3D LUT을 클라이언트에서 적용/해제하는 토글입니다. 기본값은 "적용"(보정됨)이며, 판독의가 상황에 따라 "미적용"으로 즉시 전환할 수 있습니다 ([`ARCHITECTURE.md`](../../ARCHITECTURE.md) 결정 #5).
- 이 보정들은 화면 표시에만 영향을 주며, `Annotation`의 지오메트리/측정값이나 타일 캐시 자체에는 영향을 주지 않습니다.

## 세션 / 토큰

- 뷰어 진입(슬라이드 열람) 시 backend에 토큰을 요청하며, 이때 권한 검증과 감사로그 기록이 함께 이루어집니다 ([`ARCHITECTURE.md`](../../ARCHITECTURE.md) 결정 #2).
- 슬라이드를 전환하면 새 슬라이드에 대한 토큰을 다시 요청합니다 (토큰은 슬라이드 단위로 범위가 한정됨).
- 토큰 만료 시 자동 재발급 흐름은 [`ARCHITECTURE.md`](../../ARCHITECTURE.md) 미결정 항목으로 별도 설계합니다.

## 권한별 동작

| Case 상태 | 뷰어 동작 |
|---|---|
| 접수 / 염색 | 슬라이드가 아직 준비되지 않았거나 업로드 진행 중 |
| 검토 | 전체 기능 (어노테이션 작성/수정/삭제 포함) |
| 서명완료 | 읽기 전용 — 어노테이션 도구 비활성화 (재오픈 전까지) |
