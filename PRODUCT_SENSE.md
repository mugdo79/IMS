# PRODUCT_SENSE.md

이 문서는 디지털 병리 IMS가 "누구를 위해, 어떤 워크플로우를, 어떤 기준으로 잘 동작해야 하는가"를 정리합니다.
구현 가능 여부보다 **사용자 경험/임상 워크플로우 관점에서의 판단 기준**을 다룹니다. 시스템 구조는 [`ARCHITECTURE.md`](ARCHITECTURE.md), 가치 판단의 배경은 [`docs/design-docs/core-beliefs.md`](docs/design-docs/core-beliefs.md)를 참고하세요.

## 이 제품은 무엇인가

판독의가 유리 슬라이드 대신 디지털 슬라이드(WSI)를 보고 판독·어노테이션하며, 병리사는 케이스/검체/슬라이드를 등록·추적하는 임상 워크플로우 도구입니다. AI 형태계측 결과(CellAnnotation)는 판독을 보조하는 참고 정보로 제공됩니다.

## 사용자 역할 (`User.역할` 기준)

### 판독의 (Pathologist) — 핵심 사용자

- 주 업무: 배정된 케이스의 슬라이드를 열람·판독, 어노테이션 작성, 케이스 서명(완료 처리)
- 하루에 다수 케이스를 처리 — 케이스 전환/슬라이드 전환의 속도와 클릭 수가 체감 품질에 직결됨
- 요구: 빠른 로딩과 매끄러운 줌/팬, 세포 구분이 가능한 수준의 타일 선명도, 이전 케이스 이력 참조, AI 어노테이션 참고

### 병리사 (Lab technician)

- 주 업무: 케이스 접수, 검체/블록/슬라이드 등록, 스캔된 WSI 업로드, 케이스 상태 관리(접수 → 염색 → 검토 대기)

### 관리자 (Admin)

- 주 업무: 사용자 계정/권한 관리, 감사로그 조회, 시스템 설정

## 핵심 워크플로우

> 케이스 상태 전이 규칙과 배정(담당 판독의) 방식의 상세 스펙은 [`docs/product-specs/case-status-workflow.md`](docs/product-specs/case-status-workflow.md) 참고.

### 1. 케이스 생성 ~ 슬라이드 등록 (병리사)

1. 환자/케이스 접수 (`Case` 생성, 환자 정보 입력)
2. 검체/블록 등록 (`Specimen`, `Block`)
3. 슬라이드 스캔 → WSI 업로드 → `Slide` 레코드 생성 (배율/MPP 등 메타데이터 추출)
4. `Case` 상태: 접수 → 염색 → 검토 대기

### 2. 슬라이드 판독 ~ 서명 (판독의)

1. 워크리스트에서 배정된 케이스 선택
2. WSI 뷰어로 슬라이드 열람 (줌/팬, 필요 시 여러 슬라이드 비교)
3. 필요 시 AI 셀 어노테이션(`CellAnnotation`)을 별도 레이어로 참고
4. 직접 어노테이션 작성 (`Annotation` — 영역/측정/메모)
5. 판독 완료 → `Case` 상태: 서명완료

### 3. 감사/추적 (전 과정 공통)

- 모든 슬라이드 열람·어노테이션 변경은 `AuditLog`에 기록 (규제 대응, [`ARCHITECTURE.md`](ARCHITECTURE.md) 결정 #2)

## 좋은 제품의 기준 (이 도메인의 "Product Sense")

- **신뢰성 > 기능 추가**: 환자-케이스-슬라이드 매칭 오류나 잘못된 슬라이드 표시는 치명적 오류입니다. 새 기능보다 기존 흐름의 정확성이 항상 우선합니다.
- **뷰어 반응성**: 줌/팬 시 타일 로딩 지연·끊김은 판독 흐름을 직접 방해합니다. 체감 속도가 핵심 품질 지표입니다.
- **타일 선명도**: 세포 형태 구분이 가능한 수준의 화질이 기본 요구사항입니다 ([`core-beliefs.md`](docs/design-docs/core-beliefs.md) 원칙 #1).
- **워크리스트 효율**: 판독의는 하루에 다수 케이스를 처리합니다 — 케이스 간 전환, 상태 변경에 불필요한 클릭/대기를 최소화합니다.
- **AI 보조와 판독의 판단의 분리**: AI 어노테이션은 명확히 구분되는 레이어/스타일로 표시하며, 판독의 판독을 가리거나 대체하지 않습니다 ([`core-beliefs.md`](docs/design-docs/core-beliefs.md) 원칙 #5).

## 범위

### MVP

- 케이스/검체/블록/슬라이드 관리, WSI 뷰어, 수동 어노테이션, 케이스 상태 워크플로우, 감사로그

### 향후

- AI 셀 어노테이션 자동 수집/표시
- LIS 연동
- 다기관/SSO
- 토큰 갱신 자동화 (장시간 뷰잉 세션)

## 미해결 / 추후 논의

- 다중 슬라이드 비교 뷰 (한 화면에 여러 슬라이드 동시 표시)
- 알림/협업 기능 (코멘트, 2차 판독 요청 등)
